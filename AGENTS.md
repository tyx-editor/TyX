## Commands

```bash
# Frontend dev (web only)
bun run dev

# Desktop app dev (requires Rust + Tauri)
bun run dev:app

# Build WASM converters (needed before dev if missing)
bun run predev

# Build web
bun run build

# Lint
bun run lint

# E2E tests (WebDriverIO + Tauri)
bun run test

# Rust tests
cargo test --workspace

# Rust snapshot tests (converters)
cargo insta test -p tyx-converters --accept
```

## Architecture

TyX is a WYSIWYG editor for Typst (like LyX). Two deployment targets share the same frontend: web (`app.tyx-editor.com`) and desktop (Tauri).

### Data Flow

1. User edits in Lexical WYSIWYG editor
2. Editor state serializes to `TyXDocument` JSON (Zod schemas in `src/models/`)
3. WASM converter (`crates/tyx-converters-wasm`) transforms JSON → Typst code
4. Desktop: Tauri backend compiles Typst → PDF preview / saves `.tyx` files
5. Import: `.typ` files convert via `crates/typst-to-tyx`

### Key Layers

**`src/models/`** — Zod schemas defining all document types (`TyXDocument`, `TyXNode`, `FunctionDefinition`, settings). These are the canonical data structures shared between frontend and Rust (via JSON).

**`src/components/Editor.tsx`** — Lexical editor root, mounts ~13 plugins. Each plugin in `src/components/plugins/` handles a distinct concern: math (MathLive integration), Typst code blocks, function calls, tables, images, RTL, keyboard maps.

**`src/backend/`** — Abstraction over Tauri vs. web. `tauri.ts` invokes Tauri commands; `web.tsx` provides browser-compatible stubs. `index.ts` exports the right one at build time.

**`crates/tyx-to-typst/`** — Core conversion from TyX JSON nodes to Typst text. Main logic in `src/lib.rs`: `node_to_text`, format application.

**`crates/tyx-schema/`** — Rust types matching the Zod models (generated + `TextFormat` enums). Used by converters.

**`src-tauri/src/cmds.rs`** — Tauri command handlers: open, save, saveas, PDF generation, settings, image handling. Uses bundled Typst compiler (tinymist fork).

### Build Notes

- WASM output goes to `src/converters/` (gitignored; regenerated by `bun run predev`)
- Web build: `vite.config.ts` copies static assets + WASM into `dist/`
- Tauri config: `src-tauri/tauri.conf.json` (window, updater, bundle settings)
- Workspace Cargo.toml defines shared dependency versions for all 6 crates

### Tech Stack

- Frontend: React 18, TypeScript, Lexical, Mantine, MathLive, Zod, Vite
- Desktop: Tauri 2.x, Typst 0.14 + tinymist (patched LSP), typstyle
- WASM: wasm-bindgen, wasm-pack
- Package manager: Bun
